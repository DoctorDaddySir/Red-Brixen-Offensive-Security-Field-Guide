# Assumption Breaking

## Purpose

Eliminate false beliefs that block progress.

---

## Common Dangerous Assumptions

- “This service is secure”
- “There are no credentials here”
- “This path won’t work”
- “I already checked that”

---

## Replace With

- “What have I NOT tested?”
- “What did I overlook?”
- “What is assumed to be trusted?”

---

## Techniques

### 1. Re-check Everything

- rerun enumeration
- re-open directories
- re-check services

---

### 2. Invert Thinking

Instead of:
“What works?”

Ask:
“What SHOULD NOT work?”

---

### 3. Challenge Trust Boundaries

- file permissions
- service configs
- user privileges

---

## When To Use

- stuck for 10+ minutes
- repeating same steps
- no new findings

---

## Golden Rule

Assumptions are invisible blockers.

Break them aggressively.
